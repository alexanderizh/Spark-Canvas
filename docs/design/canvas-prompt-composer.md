# 无限画布 Prompt 参数编排器

> 状态: 已落地 | 最后核对: 2026-07-15

## 产品边界

- 功能节点的内置能力提示词属于 `systemPrompt`，始终隐式，不进入用户可见编辑区。
- 用户编辑区的唯一真值是 `CanvasPromptDocument`；旧字符串 `prompt` 仅用于兼容，不参与受控编辑器反向初始化。
- 文本、剧本、分镜、角色、场景和道具等上游输入以原子引用 Tag 表示。
- 自动连接的图片、视频和音频默认由媒体输入区承载，不自动占用文本编辑区；用户仍可通过 `@` 主动插入媒体引用。
- 自定义文本、引用 Tag 和结构化参数共享同一有序文档，提交时按文档顺序编译。

## 编辑器模型

编辑器采用 Lexical。普通文字使用 TextNode，引用和参数使用自定义 inline DecoratorNode，因此浏览器原生负责选区、输入法、任意位置插入、换行、撤销重做和原子节点前后光标。

引用来源分为两类：

- `connection`：由画布物理连接自动产生。删除后写入 `suppressed: true`，表示保留连接但本节点不使用该参数；断开再重连后可重新初始化。
- `manual`：由 `@` 或添加菜单主动插入。删除时直接移除引用。

## 提交约束

- `systemPrompt`、`promptDocument`、媒体输入和模型参数分别提交，不互相拼接污染。
- 编辑器保存的是“可见文档”；提交编译器会基于当前有效 `inputNodeIds` 生成一次性的“可执行文档”。媒体选择器中的图片、视频、音频只在可执行文档中补为引用，因此能进入模型请求但不会回填编辑器。
- 右键创建时预绑定输入、创建后物理连线、`@` 手动插入均调用同一套文档初始化与连接对账逻辑。连线只改变引用块和输入清单，不再继承或展开上级任务的旧 `prompt` 字符串。
- `promptDocument` 中被抑制或断开的引用不进入有效任务输入。
- `@` 新增的有效引用会进入任务输入节点清单和 relation manifest，保证运行历史可追踪。
- 文本/剧本/角色等引用编译进 `compiledUserText`；图片等媒体编译进带稳定 role 的 `inputFiles`；结构化参数编译进用户消息；`systemPrompt` 始终保持单独字段。文本模型接收 system + user + vision 输入，媒体模型适配器接收最终 prompt + typed input files。
- 引用操作节点或分组时，提交阶段会解析为其当前主产物/集合产物并冻结 `inputSnapshots`，而编辑器仍保留原始上游节点卡片，避免提交后卡片被产物正文替换。
- 任务运行或特殊工作流回绑节点时，`task.prompt` 只保存编译后的执行快照，`node.data.promptDocument` 仍保存用户可见文档，`node.data.systemPrompt` 保存隐藏指令；三者不得互相反向初始化。
- 旧节点中混合保存的功能提示词与上游正文会在面板初始化时迁移：功能指令转入隐藏层，上游文本转成连接 Tag，自动媒体 Tag 转移到媒体输入区。

## 设计参考

- Dify：区分 System/User 消息和工作流变量。
- n8n：将上游数据映射为节点参数引用。
- Lexical/Tiptap：Mention 采用可序列化、可整块删除的原子节点。
- React Flow：不同输入类型使用明确连接语义，而不是把所有输入压成字符串。
